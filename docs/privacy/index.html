qFT8 Privacy Policy

This policy outlines how qFT8 handles your data, permissions, and communications. Our goal is transparency and simplicity.

Key Principle: This app does not submit any information to a server dedicated to this app. All data processing happens locally on your device or limited (callsign, optionally approximate location, date and time) via direct connections you authorize (e.g. to QRZ with a QRZ API Key, to PSKReporter if you enable the option, or via the integrated option for web server for remote control).

üÜî User Identity

You must provide an amateur radio Callsign. This is required for the fundamental operation of the app.

  • This callsign personally identifies you.
  • It is broadcast via Radio Frequency (RF) when connected to a QMX transceiver.
  • It is encoded in audio signals for mock functionality when not connected to a transceiver.

üìç Location Data

Optionally, you can grant Approximate Location permission.

  • This is used solely to calculate your Maidenhead Grid Locator (e.g., "JN47").
  • If enabled, this coarse location (precision of 4 characters, approx. 100x100km) is shared publicly via RF transmissions as part of standard amateur radio protocols.

üéôÔ∏è Audio Permission

The app requires audio permission to capture signals.

  • Source: From the microphone (for mock/demo mode) or directly from a connected QMX transceiver via USB audio.
  • Privacy: This audio is processed in real-time to decode FT8 signals. It is never recorded, stored, or shared.

üì° Transmissions

The app emits audio signals to communicate:

  • Speaker: Used for mock functionality when no radio is connected.
  • RF (Radio Frequency): When connected to a QMX transceiver, the app instructs the radio to transmit.
  • Transmissions consist of single-tone signals containing your Callsign and Approximate Location (Grid).

üíª Optional Remote Control

The app includes an optional integrated web server for remote control within your local network.

  • Functionality: Allows you to remotely control the app and observe real-time data, including received callsigns, QSOs made, and decoding status.
  • Access: When enabled, this feature opens a local web interface accessible to other devices on your local network (WiFi/LAN).
  • Privacy: This feature is disabled by default and must be explicitly enabled in settings.

‚òÅÔ∏è Optional External Services

You may choose to connect to third-party amateur radio services. Note: Data is transmitted to these services unencrypted.

  • QRZ Logbook: If you provide an API Key, finished contacts are synced to your QRZ Logbook. Shared data includes: Callsign, Grid, Date, and Time.
  • PSKReporter: If enabled, the app reports "heard" stations to PSKReporter.info. This report includes the data you received, plus your Callsign and Grid location.

üìÇ Data Management

You can import and export your logs and settings to/from files on your device at any time.
